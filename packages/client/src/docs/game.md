## Документация игры Memo

#### Общее описание

Memo - классическая игра на развитие памяти, где игрок должен находить пары одинаковых карточек. Игра реализована на React с использованием Canvas для отрисовки игрового поля.

---

#### Игровые механики

##### Основной геймплей

- Игровое поле состоит из сетки карточек (по умолчанию 4×4)
- Все карточки изначально перевернуты рубашкой вверх
- Игрок последовательно открывает по две карточки
- Если карточки совпадают - они остаются открытыми
- Если не совпадают - переворачиваются обратно
- Игра завершается, когда все пары найдены

##### Правила

1. Начало игры: карточки перемешиваются в случайном порядке
2. Ход игрока: клик по закрытой карточке
3. Проверка совпадения:
   - Открыто 2 карточки → проверка цвета
   - Совпадение → карточки остаются открытыми
   - Несовпадение → автопереворот через 1 секунду
4. Блокировка интерфейса: во время проверки нельзя делать новые ходы
5. Победа: все пары найдены

#### Архитектура приложения

##### Структура хуков

```text

useGame (главный хук)
├── useBoard (управление доской)
│   └── useColors (управление цветами)
└── useGameLogic (игровая логика)
```

##### Детальное описание компонентов

`useColors`
Назначение: управление цветовой палитрой карточек
Функциональность:

- Хранение массива цветов
- Генерация цветов для заданного количества пар
- Возврат среза цветов по необходимости

`useBoard`
Назначение: создание и управление игровой доской
Функциональность:

- Инициализация карточек с координатами
- Расчет позиций на основе размеров и отступов
- Перемешивание карточек
- Сброс доски для новой игры
  Структура карточки:

```js
{
	id: number, // Уникальный идентификатор
	color: string, // Цвет карточки
	x: number, // X-координата на canvas
	y: number // Y-координата на canvas
}
```

`useGameLogic`
Назначение: управление состоянием игры и логикой ходов

```js

// Состояния
flipped: number[] // ID открытых карточек
matched: number[] // ID совпавших карточек
disabled: boolean // Блокировка интерфейса

// Основные функции
handleCardClick() // Обработка клика по карточке
checkMatch() // Проверка совпадения карточек
resetGame() // Сброс игрового состояния
isGameComplete() // Проверка завершения игры
```

`useGame` (главный хук)
Назначение: координация всех под-систем игры

```js

// Конфигурация
{
	rows: number, // Количество строк
	cols: number, // Количество столбцов
	cardWidth: number, // Ширина карточки
	cardHeight: number, // Высота карточки
	spacing: number // Отступ между карточками
}

// Возвращаемое значение
- Все состояния и методы дочерних хуков
- Размеры canvas
- Флаг завершения игры
```

`GameCanvas Component`
Назначение: отрисовка игрового поля и обработка взаимодействий
Функциональность:

- Отрисовка карточек на Canvas
- Обработка кликов с преобразованием координат
- Визуализация состояний (открыта/закрыта/совпала)
- Мемоизация для оптимизации перерисовок

```js
const SETTINGS = {

  cardWidth: 80, // Ширина карточки (px)

```

#### Возможности расширения

1. Таймер игры - отсчет времени с начала партии
2. Система очков - начисление очков за быстрые совпадения
3. Уровни сложности - разные размеры сетки (4×4, 6×6, 8×8)
4. Темы оформления - скины для карточек
5. Анимации - плавные перевороты карточек
6. Звуковые эффекты - звуки переворота, совпадения, победы

---

#### Преимущества архитектуры

**Модульность**

- Каждый хук отвечает за свою зону ответственности
- Легкая замена или модификация отдельных частей
- Независимое тестирование компонентов

**Масштабируемость**

- Простое добавление новых функций через хуки
- Гибкая конфигурация игры
- Возможность переиспользования хуков в других проектах

**Производительность**

- Мемоизация компонентов и функций
- Оптимизированные перерисовки Canvas
- Эффективное управление состоянием
